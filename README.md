# AI English Learning Hub

一个面向「输入 + 输出 + 复习 + 反馈」全流程的 AI 英语学习网站。
通过外刊阅读、AI 讲解、单词复习、口语对话、写作批改和学习统计，形成可持续的日常学习闭环。

> 适合有自驱力的英语学习者：希望把碎片化学习整合成体系，并且能长期跟踪进步。

---

## 1. 网站能做什么（功能总览）

### 1.1 仪表盘（首页）
- 查看今日学习进度与关键数据。
- 快速跳转到当日最重要的任务（计划、阅读、复习、输出练习）。
- 获取推荐内容，减少“今天学什么”的决策成本。

### 1.2 学习计划（Plan）
- 根据你的学习目标与每日时长自动生成当天任务。
- 任务可逐条完成，形成清晰的执行路径。
- 支持每天重新生成，让计划动态适配你的状态。

### 1.3 外刊发现（Discover）
- 从 RSS 源拉取最新英文文章，构建可持续输入。
- 可按难度、分类筛选，支持按主题抓取内容。
- 适合做“每日精读/泛读素材池”。

### 1.4 阅读器（Reader）
- 进入文章后可进行沉浸式阅读。
- 支持中英双语切换（按句翻译）。
- 支持划词交互：
  - 选中词/短语快速翻译；
  - 获取 AI 解释与语境说明；
  - 一键加入生词本。
- 让“读懂 + 学会 + 记住”在同一页面完成。

### 1.5 生词本与复习（Vocab）
- 管理你在阅读中积累的词汇。
- 提供今日复习清单，按学习节奏推进。
- 基于间隔重复（SM-2）安排复习，帮助长期记忆而非短期记忆。

### 1.6 词汇书（Wordbook）
- 维护主题词汇书（可用于专项突破）。
- 可结合词汇书生成练习内容，强化“词汇→语境”的迁移。
- 便于备考或按行业领域（如科技、商业）做定向积累。

### 1.7 口语陪练（Speaking）
- 提供多场景对话训练（如日常交流、面试等）。
- 你输入英文后，AI 会继续对话并给出表达优化建议。
- 在“说出来”的过程中进行即时纠错，提升表达自然度。

### 1.8 写作批改（Writing）
- 提交英文作文后获得 AI 反馈。
- 支持语法纠错、表达优化、结构建议与评分参考。
- 帮你把“知道”转化为“写得出来、写得更好”。

### 1.9 学习统计（Stats）
- 查看近 7 天学习表现。
- 汇总任务完成、词汇学习、复习完成率等指标。
- 用数据识别薄弱环节，指导下一阶段学习策略。

### 1.10 设置（Settings）
- 配置 API Key、模型、基础地址等 AI 连接参数。
- 设置个人学习目标与每日学习时长。
- 支持数据导入导出与 WebDAV 同步（便于备份/跨设备）。

## 2. 典型学习流程（推荐）

1. 在 **Settings** 中完成 API 与学习目标配置。
2. 到 **Plan** 生成今日任务。
3. 在 **Discover** 抓取文章并选择合适难度。
4. 使用 **Reader** 阅读 + 划词 + AI 讲解，沉淀到 **Vocab**。
5. 在 **Vocab** 完成今日复习。
6. 用 **Speaking / Writing** 做输出训练。
7. 在 **Stats** 复盘今日与近 7 天学习效果。

这个流程可帮助你形成：
**输入（阅读）→ 理解（翻译/讲解）→ 记忆（复习）→ 输出（口语/写作）→ 反馈（统计）**。

## 3. 技术特点

- **前端框架**：Next.js 14 + TypeScript + Tailwind CSS
- **运行模式**：静态导出，可部署到 GitHub Pages
- **数据存储**：IndexedDB（Dexie.js，本地持久化）
- **AI 调用**：浏览器直连 OpenAI 兼容 API
- **内容来源**：RSS 抓取（通过代理方案处理跨域）

> 隐私说明：API Key 与学习数据默认存储在本地浏览器，不依赖中心化后端数据库。

## 4. 快速开始

### 4.1 本地开发

```bash
cd frontend
npm install
npm run dev
```

打开：`http://localhost:3000`

### 4.2 首次使用配置

1. 打开网站后进入 **Settings**。
2. 填入你的 AI API Key（支持 OpenAI 兼容接口）。
3. 设置学习目标与每日时长。
4. 前往 **Discover** 点击 `Fetch New` 抓取文章。
5. 开始你的每日学习流程。

## 5. 部署到 GitHub Pages

### 方式一：GitHub Actions 自动部署（推荐）

1. 将项目推送到 GitHub 仓库。
2. 进入仓库 `Settings → Pages → Source`，选择 **GitHub Actions**。
3. 之后每次 push 到 `main` 分支会自动构建并部署。

如果仓库名不是 `username.github.io`，请在 `frontend/next.config.js` 设置 `basePath`：

```js
const nextConfig = {
  output: "export",
  images: { unoptimized: true },
  basePath: "/你的仓库名", // 取消注释并修改
  trailingSlash: true,
};
```

### 方式二：手动构建部署

```bash
cd frontend
npm install
npx next build
# 将 frontend/out/ 目录内容部署到 GitHub Pages
```

## 6. 页面路由

| 路径 | 页面说明 |
|------|----------|
| `/` | 仪表盘（学习总览） |
| `/plan` | 今日学习计划 |

| `/reader?id=N` | 阅读器（划词、翻译、讲解） |

| `/writing` | 写作批改 |
| `/stats` | 学习统计 |

## 7. 设计风格

整体采用 **Swiss Style（瑞士国际风格）**：
- 强调网格系统与秩序感；
- 黑白主色，突出信息层级；
- 大量留白，降低认知负担；
- 高可读性字体与简洁交互。
